# Claude Code – כללי עבודה לפרויקט

## זרימת עבודה עם Git

### ענפים (Branches)
- **כל פיצ'ר או תיקון** חייב להיפתח על ברנצ' נפרד — אסור לעבוד ישירות על `main`.
- שמות ברנצ'ים: `fix/<תיאור-קצר>` לתיקונים, `feat/<תיאור-קצר>` לפיצ'רים.
  - לדוגמה: `fix/use-gender-text-reference-error`, `feat/shopping-list`

### Push ו-Pull Request
- לאחר סיום כל שינוי — **פתח PR אוטומטית** לעיון המשתמש.
- אל תמזג ל-`main` לעולם — המיזוג יעשה על ידי המשתמש בלבד לאחר אישורו.
- הכלל הזה תקף גם לתיקוני באגים קטנים.

### תהליך מלא לכל משימה
1. צור ברנצ' חדש מ-`main`.
2. בצע את השינויים.
3. עשה commit עם הודעה ברורה.
4. דחוף (`push`) את הברנצ' ל-GitHub.
5. פתח PR עם תיאור קצר של מה השתנה ולמה.
6. **עצור — המשתמש יאשר ויעשה מיזוג.**

## חיסכון בעלויות Netlify
- **לא לפתוח PR על כל שינוי קטן.** לצבור שינויים קשורים ל-commit/PR אחד כשאפשר.
- שינויים שלא משפיעים על הקוד (README, תיעוד, הגדרות לוקליות) — לא צריכים PR נפרד שיגרום לדיפלוי.
- קובץ `netlify.toml` מוגדר עם `ignore` — דיפלוי יקרה רק כששונו קבצים רלוונטיים (`public/`, `src/`, `netlify/`, `package.json`, `netlify.toml`).

## שפה ב-Git
- **כל מה שעולה ל-GitHub חייב להיות באנגלית בלבד** — commit messages, branch names, PR titles, PR descriptions.
- עברית מותרת רק בתוך הקוד עצמו (טקסטים של UI, קומנטרים פנימיים) ובתקשורת עם המשתמש.

## מה אסור
- לא לדחוף ישירות ל-`main` (אף פעם).
- לא למזג PR ללא אישור מפורש.
- לא להשתמש ב-`--force` על `main`.
